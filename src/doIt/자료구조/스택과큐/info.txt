 * stack : 삽입과 삭제 연산이 후입선출(Last-in First-out)로 이뤄지는 자료구조
  [스택 용어]
    위치
   - top : 삽입과 삭젝가 일어나는 위치를 뜻함.
   연산
   - push : top 위치에 새로운 데이터를 삽입하는 연산
   - pop : top 위치에 현재 있는 데이터를 삭제하고 확인하는 연산
   - peek : top 위치에 현재 있는 데이터를 단순 확인하는 연산

   스택은 우선 탐색(DFS: Depth First Search), 백트래킹 종류의 코테에 효과적, 후입선출은 개념 자체가 재귀 함수 알고리즘 원리와 일맥상통

 * Queue : 삽입과 삭제 연산이 선입선출(First-in First-out)로 이뤄지는 자료구조
 [큐 용어]
    - rear : 큐에서 가장 끝 데이터를 가리키는 영역
    - front : 큐에서 가장 앞의 데이터를 가리키는 영역
    - add : rear 부분에 새로운 데이터를 삽입하는 연산
    - poll : front 부분에 있는 데이터를 삭제하고 확인하는 연산
    - peek : 큐의 맨앞에 있는 데이터를 확인할 떼 사용하는 연산

    큐는 너비 우선 탐색(BFS : Breadth First Search)에 자주 사용 함
